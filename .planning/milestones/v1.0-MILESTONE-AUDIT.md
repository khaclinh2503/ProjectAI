## Tech Debt Inventory

### Phase 3 — Process Gap
- **Missing VERIFICATION.md** — Phase was completed via human checkpoint in 03-03-SUMMARY. No automated verifier ran against Phase 3 artifacts independently. All 5 Phase 3 requirements confirmed passing in SUMMARY, but no structured VERIFICATION.md produced.

### Phase 5 — Logic Duplication
- **getUrgencyStage() unused** — GameController._updateTimerUrgency() reimplements the same 4-stage boundary logic inline. If thresholds change, two places must be updated. The tested helper is dead code in production.
- **getMilestoneLabel() unused** — GameController._checkMilestone() reimplements the threshold loop inline. Same maintenance risk.
- **MILESTONE_THRESHOLDS unused transitively** — follows from getMilestoneLabel() being unused.
- **HUD-03 display discrepancy** — comboLabel shows `Combo x{tapCount}` (blank when tapCount ≤ 1). Phase 05-02-SUMMARY documented intention to show `Combo x{multiplier.toFixed(1)}` always (visible at x1.0 from session start). Current code does not implement this. Multiplier is correctly applied to scoring (GAME-04 unaffected); only the label format differs from documentation.

### Phase 2 — Unused Public API
- **ComboSystem.applyToScore() never called** — GameState.applyCorrectTap() performs `Math.round(rawScore * combo.multiplier)` directly. This method is a public API surface with no callers.

### Phase 3 — Orphaned Test
