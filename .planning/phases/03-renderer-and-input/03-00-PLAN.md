<feature>
  <name>GameState — scoring model</name>
  <files>BloomTap/assets/scripts/logic/GameState.ts, BloomTap/assets/scripts/logic/GameState.test.ts</files>
  <behavior>
    - GameState has `score: number` (starts 0), `sessionStartMs: number`
    - `reset()` sets score=0, sessionStartMs=performance.now()
    - `applyCorrectTap(rawScore, combo)` adds Math.round(rawScore * combo.multiplier) to score, then calls combo.onCorrectTap()
    - `applyWrongTap(combo)` subtracts WRONG_TAP_PENALTY from score (floor at 0 is NOT applied — score can go negative), then calls combo.onWrongTap()
    - `getElapsedMs()` returns performance.now() - sessionStartMs
    - WRONG_TAP_PENALTY is exported constant = 10
    - GAME-01: after applyCorrectTap with rawScore=80, multiplier=1 → score=80
    - GAME-02: FULL_BLOOM rawScore (e.g. 120) > BLOOMING rawScore (e.g. 80) for CHERRY → applyCorrectTap(120,combo) gives score=120 > score=80
    - GAME-03: applyWrongTap → score decreases by 10; combo.multiplier becomes 1
    - Multiple correct taps accumulate: applyCorrectTap(80,combo) × 2 with rising multiplier → score > 160
    - After applyWrongTap, combo.multiplier is 1 (verified via combo.multiplier getter)
  </behavior>
  <implementation>
    Create GameState.ts as plain TypeScript with no `cc` imports. Export `WRONG_TAP_PENALTY = 10` as a named constant alongside the class. The class receives a `ComboSystem` reference via method parameters (not constructor injection) so tests can pass a fresh ComboSystem per test case without resetting. The `applyCorrectTap(rawScore: number, combo: ComboSystem)` signature keeps the GameState stateless regarding combo — combo is owned by GameController in Phase 3 wiring. Apply `Math.round()` to the final score delta to avoid floating-point accumulation (e.g. 80 * 1.5 = 120, not 119.99999).
