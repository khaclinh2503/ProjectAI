# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Results and Persistence.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> NOTE: This is a Cocos Creator game project (TypeScript + Cocos Creator 3.8.x).
> There is no CSS, no web component library, and no shadcn. All design tokens
> below map to Cocos Creator Label properties, Node transforms, and Color objects.
> "Spacing" means Node layout offsets in Cocos units (≈ px at 720x1280 design resolution).

## Typography

All sizes are Cocos Creator Label `fontSize` values in Cocos units (design resolution 720x1280).
Established pattern from Phase 4 HUD — extended here to results overlay.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 72 | Bold (700) | 1.2 | `finalScoreLabel` — session score (existing, dominant) |
| Heading | 48 | Bold (700) | 1.2 | "NEW BEST!" celebration label (`newBestLabel`) |
| Body | 32 | Regular (400) | 1.5 | Highscore row, Best combo row, Accuracy row |
| Label | 24 | Regular (400) | 1.5 | Restart button text |

Two weights only: Regular (400) for stat rows and button; Bold (700) for score and celebration.

Source: CONTEXT.md §1 — "Score — large, visually dominant"; §3 — "NEW BEST! Label appears above score row — yellow color, scale-pop tween."
Default applied for exact sizes — consistent with Phase 4 HUD sizing pattern (HUD score label visible at 72px in existing GameScene).

## Color

All values are Cocos Creator `Color(R, G, B, A)` objects. Hex equivalents shown for reference.
Palette is locked by prior phases (FlowerColors.ts + GameController.ts color constants).

| Role | Hex | Cocos Color | Usage |
|------|-----|-------------|-------|
| Dominant (60%) | #000000 / deep dark | Color(0, 0, 0, 200) | gameOverOverlay background panel (semi-transparent) |
| Secondary (30%) | #1A1A2E | Color(26, 26, 46, 255) | Overlay inner content area, existing scene background |
| Accent (10%) | #FFDC32 | Color(255, 220, 50, 255) | "NEW BEST!" label text ONLY |
| Destructive | #DC3232 | Color(220, 50, 50, 255) | Wrong-flash overlay (established Phase 5 — not used in results screen) |

Accent reserved for: the `newBestLabel` text color exclusively.
This matches the existing `TIMER_COLOR_URGENCY1` yellow (#FFDC32) already in `GameController.ts`.

Stat row labels (Highscore, Best combo, Accuracy) use: Color(255, 255, 255, 255) — white, matching existing HUD label color.
`finalScoreLabel` uses: Color(255, 255, 255, 255) — white (existing, unchanged).

Source: CONTEXT.md §3 — "yellow color, scale-pop tween"; GameController.ts — `TIMER_COLOR_URGENCY1 = new Color(255, 220, 50, 255)`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Play Again" — on `restartButton` |
| Score label | `"Score: {N}"` — e.g. `"Score: 1240"` (existing `finalScoreLabel` format, unchanged) |
| Highscore label | `"Highscore: {N}"` — e.g. `"Highscore: 1200"` (locked — CONTEXT.md §5) |
| Best combo label | `"Best combo: {N}"` — e.g. `"Best combo: 7"` (integer streak, not multiplier — CONTEXT.md §1) |
| Accuracy label | `"{X} / {Y} taps ({Z}%)"` — e.g. `"18 / 24 taps (75%)"` (CONTEXT.md §1) |
| Celebration label | `"NEW BEST!"` — all caps, no trailing punctuation (CONTEXT.md §3) |
| Empty state — no highscore | `highscoreLabel.node.active = false` — row is hidden entirely, no placeholder text (CONTEXT.md §5) |
| Zero-tap edge case | Accuracy displays `"0 / 0 taps (0%)"` — guard: `totalTaps > 0 ? Math.round(...) : 0` (RESEARCH.md Pitfall 5) |

No destructive actions in Phase 6. Restart is non-destructive (in-place reset, locked Phase 4 decision).
No confirmation dialog required.

Source: CONTEXT.md §1 — label text formats; §3 — "NEW BEST!"; §5 — "Highscore: 1200".

## Interaction Contract

### Game-Over Overlay Sequence

| Step | Timing | Visual |
|------|--------|--------|
| 1 | t=0 (timer hits 0) | `gameOverOverlay.active = true`; `finalScoreLabel` shows `"Score: N"` immediately |
| 2 | t=0 | Stat rows appear: `highscoreLabel` (if stored), `bestComboLabel`, `accuracyLabel` |
| 3 | t=+0.5s (only if new best) | `newBestLabel.node.active = true`; scale-pop tween plays |
| 4 | Persistent | "Play Again" button active and tappable |

### "NEW BEST!" Tween Specification

| Property | Value |
|----------|-------|
| Delay | 0.5s via `scheduleOnce` |
| Initial scale | Vec3(0, 0, 1) — hidden, not opacity |
| Step 1 | `to(0.15s, Vec3(1.2, 1.2, 1), easing: 'backOut')` — overshoot |
| Step 2 | `to(0.10s, Vec3(1.0, 1.0, 1), easing: 'cubicOut')` — settle |
| Duration total | 250ms animation after 500ms delay |
| Auto-hide | No — stays visible until restart |

Source: CONTEXT.md §3 — exact tween values.

### Highscore Node Visibility Rules

| Condition | `highscoreLabel.node.active` | Label text |
|-----------|------------------------------|------------|
| `StorageService.get('highscore') === null` AND `finalScore <= 0` | false | — |
| `StorageService.get('highscore') === null` AND `finalScore > 0` | true (revealed on first beat) | `"Highscore: {finalScore}"` |
| Stored highscore exists, no new best | true | `"Highscore: {stored}"` |
| New best (`finalScore > currentBest`) | true | `"Highscore: {finalScore}"` |

Rule: Use `node.active = false`, NOT `opacity = 0`. Active=false prevents layout space reservation.

Source: CONTEXT.md §5 — locked decision.

### Restart Behavior

On "Play Again" tap:
1. `newBestLabel.node.active = false` (RESEARCH.md Pitfall 4 — must hide explicitly)
2. `gameState.correctTaps`, `wrongTaps`, `peakStreak` zeroed in `onRestartTapped()`
3. `gameOverOverlay.active = false`
4. Flow returns to start screen — countdown → new session
