## Phase Scope: What Changes Visually

Phase 8 introduces exactly one observable visual change:

**Before (current):** Game board is empty for the entire 3-second countdown after the player taps Start. Flowers only appear when the PLAYING phase begins.

**After (phase 8):** Flowers appear on the board the moment the player taps Start — the board is populated with `initialCount` flowers while the 3-second countdown (3→2→1) runs. The countdown overlay itself is unchanged.

No new UI components, screens, overlays, or layouts are introduced. All existing visual elements (countdown overlay, HUD, game board, cells, flower states) remain identical in appearance.

## Behavioral Contract (Phase 8 specific)

This section replaces the visual component inventory typically found in UI specs, because Phase 8 changes behavior only.

### State Transition: Start Tap

| Moment | Board State | Countdown Overlay | Input State |
|--------|-------------|-------------------|-------------|
| Before Start tap | Empty (0 flowers) | Not visible | Start button active |
| Immediately after Start tap | `initialCount` flowers visible (from `spawnPhases[0]`) | Begins: "3" | Locked — player cannot tap flowers |
| Countdown tick "2" | Flowers cycling (may advance state per FSM) | Shows "2" | Locked |
| Countdown tick "1" | Flowers cycling | Shows "1" | Locked |
| PLAYING begins | Flowers alive + regular spawn cycle resumes | Hidden | Active — player can tap |

**Source (D-01, D-02):** CONTEXT.md locked decisions.

### Initial Burst Constraints

| Constraint | Value | Source |
|------------|-------|--------|
| Flower count | `initialCount` from `flowers.json spawnPhases[0]` (recommended: 5) | D-03, RESEARCH.md Open Question 1 |
| Flower type selection | Weighted random using `spawnPhases[0]` weights | D-05 |
| Maximum alive cap | `maxAlive` from `spawnPhases[0]` (8) — burst stops early if cap reached | D-06 |
| Grid state at burst time | Always empty (restart path calls `grid.clearAll()` before returning to start screen) | RESEARCH.md Pitfall 1 |

### What the Player Sees (Observable)

1. Player taps the Start button.
2. Within the same frame, flowers appear on the game board (no empty board period).
3. The 3→2→1 countdown overlay begins animating on top of the populated board.
4. Flowers are visible and cycling through their FSM states during the countdown.
5. Player cannot tap flowers during the countdown (input lock unchanged).
6. When PLAYING begins, flowers already on the board are "aged" by ~3 seconds — some may be in BLOOMING or later states. This is intended behavior, not a bug.

### Interaction Lock

No new interaction patterns. The input lock during countdown is unchanged from current behavior (D-02). No new buttons, gestures, or interaction surfaces are added.
