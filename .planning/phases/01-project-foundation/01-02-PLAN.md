  <files>assets/scripts/BootController.ts, assets/scripts/GameController.ts</files>
  <action>
    Write two TypeScript component files in assets/scripts/.

    --- assets/scripts/BootController.ts ---
    BootScene's one job: immediately transition to GameScene. Nothing else.
    - Import `_decorator`, `Component`, `director` from 'cc'
    - `@ccclass('BootController')` decorator
    - `onLoad(): void` calls `director.loadScene('GameScene')`
    - No update(), no start(), no other logic

    ```typescript
    import { _decorator, Component, director } from 'cc';
    const { ccclass } = _decorator;

    @ccclass('BootController')
    export class BootController extends Component {
        onLoad(): void {
            director.loadScene('GameScene');
        }
    }
    ```

    --- assets/scripts/GameController.ts ---
    Phase 1 placeholder for GameScene root node. Sets the title label text.
    - Import `_decorator`, `Component`, `Label` from 'cc'
    - `@ccclass('GameController')` and `@property` decorators
    - `@property(Label) titleLabel: Label | null = null` for Inspector wiring
    - `onLoad(): void` sets `this.titleLabel.string = 'Bloom Tap'` (null-guarded)
    - No game logic — this component expands in Phase 3

    ```typescript
    import { _decorator, Component, Label } from 'cc';
    const { ccclass, property } = _decorator;

    @ccclass('GameController')
    export class GameController extends Component {
        @property(Label)
        titleLabel: Label | null = null;

        onLoad(): void {
            if (this.titleLabel) {
                this.titleLabel.string = 'Bloom Tap';
            }
        }
    }
    ```

    5. Save scene (Ctrl+S)

    1. Menu → Project → Build (or Build & Publish)

<task type="auto">
  <name>Task 2: Create build template with touch-action:none CSS</name>
  <files>build-templates/web-mobile/index.html</files>
  <action>
    Create the directory build-templates/web-mobile/ at the project root and write a custom HTML build template.

    Cocos Creator uses this template when generating web-mobile build output. The critical purpose is `touch-action: none` on the #GameCanvas element — without it, vertical touch drags trigger browser page scroll instead of game input on Android Chrome and mobile Safari (STATE.md Critical Pitfall #5).

    The build template is picked up automatically by Cocos Creator during "Build & Publish → Web Mobile". It is NOT used during editor preview (Ctrl+P) — only in actual builds.

    Write this file at E:/workspace/ProjectAI/build-templates/web-mobile/index.html:

    ```html
    <!DOCTYPE html>
    <html>
    <head>
        <meta charset="utf-8">
        <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">
        <meta name="apple-mobile-web-app-capable" content="yes">
        <meta name="apple-mobile-web-app-status-bar-style" content="black">
        <title>Bloom Tap</title>
        <style>
            html, body {
                margin: 0;
                padding: 0;
                overflow: hidden;
                height: 100%;
                background: #000000;
            }
            #GameCanvas {
                touch-action: none;
                -webkit-user-select: none;
                user-select: none;
                display: block;
            }
        </style>
    </head>
    <body>
        <canvas id="GameCanvas"></canvas>
    </body>
    </html>
    ```

    The build-templates/ folder must be at E:/workspace/ProjectAI/build-templates/ (project root), NOT inside assets/. Cocos Creator looks for build templates at the project root level.

    The Cocos build system injects its scripts into this template during build. The CSS rules are what matter here — `touch-action: none` and `overflow: hidden`.
  </action>
