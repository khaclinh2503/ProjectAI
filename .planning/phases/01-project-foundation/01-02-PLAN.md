Replace any temporary placeholder with the full BootScene implementation. This file satisfies two locked decisions from CONTEXT.md: (1) text-only splash with "Bloom Tap" + "Tap to Start", (2) pointerdown handler that unlocks AudioContext and triggers the 300ms fade-out transition.

```typescript
import Phaser from 'phaser';

export class BootScene extends Phaser.Scene {
  constructor() {
    super({ key: 'BootScene' });
  }

  create(): void {
    const { width, height } = this.scale;

    // Title text — centered horizontally at 40% from top
    this.add.text(width / 2, height * 0.4, 'Bloom Tap', {
      fontSize: '48px',
      color: '#ffffff',
      fontStyle: 'bold',
    }).setOrigin(0.5);

    // Tap prompt text — centered at 60% from top
    this.add.text(width / 2, height * 0.6, 'Tap to Start', {
      fontSize: '24px',
      color: '#aaaaaa',
    }).setOrigin(0.5);

    // Single tap handler via pointerdown — NOT touchend/touchstart/click.
    // This is the ONLY user gesture before the game loop begins.
    // It serves double duty: transition trigger AND iOS Safari AudioContext unlock.
    this.input.once('pointerdown', () => {
      // iOS Safari requires a user gesture before AudioContext can play audio.
      // Phase 5 adds sound effects — if this is skipped, all audio will be
      // silent on iOS Safari.
      if (this.sound.context && this.sound.context.state === 'suspended') {
        this.sound.context.resume();
      }

      // Fade out to black over 300ms, then start GameScene.
      this.cameras.main.fadeOut(300, 0, 0, 0);
    });

    // Start GameScene after fade completes — do NOT start it immediately
    // or during the fade (FADE_OUT_COMPLETE fires after 300ms).
    this.cameras.main.once(
      Phaser.Cameras.Scene2D.Events.FADE_OUT_COMPLETE,
      () => {
        this.scene.start('GameScene');
      }
    );
  }
}
```

Do NOT use:
- `touchend`, `touchstart`, or `click` events — 100-300ms latency on mobile
- `this.input.on` for the tap-to-start (use `once` — single-fire, no duplicate transitions)
- Manual setTimeout for the transition — use the FADE_OUT_COMPLETE event
