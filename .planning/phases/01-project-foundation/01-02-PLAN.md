    **Step 2 — Write assets/core/scripts/adapters/IPlatformAdapter.ts.**

    This is the contract. Every method here maps to a platform capability used by game code. Phase 2 (SaveSystem) will add save/load methods to this interface — leave a comment indicating the extension point.

    ```typescript
    // IPlatformAdapter.ts — Platform capability contract
    // ALL external SDK calls must go through this interface.
    // Game logic imports this interface only — never the concrete implementations.
    //
    // Extension points for future phases:
    //   Phase 2 (TECH-05): add getPlayerDataAsync / setPlayerDataAsync
    //   Phase 6 (JUICE-02): add playSound or audio unlock if platform-specific
    //
    // ANTI-PATTERN to avoid: importing FBInstant, firebase, or admob
    //   anywhere outside assets/core/scripts/adapters/FBInstantAdapter.ts

    export interface IPlatformAdapter {
        /**
         * Must be called first. Resolves when the platform is ready.
         * FB Instant: calls FBInstant.initializeAsync()
         * Native / null: resolves immediately
         */
        initialize(): Promise<void>;

        /**
         * Reports loading progress to the platform (0–100).
         * FB Instant: calls FBInstant.setLoadingProgress(percent)
         * Native / null: no-op
         */
        setLoadingProgress(percent: number): void;

        /**
         * Signals the platform that the game is ready to start.
         * FB Instant: calls FBInstant.startGameAsync()
         * Native / null: resolves immediately
         * MUST be called after setLoadingProgress(100) and before any gameplay logic.
         */
        startGame(): Promise<void>;

        /**
         * Returns the platform player ID, or null if not available.
         * FB Instant: FBInstant.player.getID()
         * Native / null: returns 'dev-player'
         */
        getPlayerID(): string | null;
    }
    ```

    **Step 4 — Write assets/core/scripts/adapters/FBInstantAdapter.ts.**

    This is the ONLY file in the entire project permitted to reference `FBInstant.*`. Do not change this constraint.

    ```typescript
    // FBInstantAdapter.ts — FB Instant Games SDK wrapper
    // ONLY FILE ALLOWED to reference the FBInstant global.
    // The FBInstant global is injected by Creator's fb-instant-games build template.
    // Type declarations live in assets/fbinstant.d.ts.
    //
    // Selected by: PlatformDetector.createPlatformAdapter() when FBInstant is present at runtime.
    //
    // Init sequence (MUST follow this order — see RESEARCH.md Pitfall 4):
    //   1. initializeAsync()   ← before ANY other FBInstant call
    //   2. setLoadingProgress() during asset loading
    //   3. startGameAsync()    ← after setLoadingProgress(100), before gameplay

    import { IPlatformAdapter } from './IPlatformAdapter';

    export class FBInstantAdapter implements IPlatformAdapter {
        async initialize(): Promise<void> {
            try {
                await FBInstant.initializeAsync();
            } catch (err) {
                console.error('[FBInstantAdapter] initializeAsync failed:', err);
                throw err;
            }
        }

        setLoadingProgress(percent: number): void {
            // Clamp to 0–100 as FB SDK may reject out-of-range values
            const clamped = Math.min(100, Math.max(0, Math.round(percent)));
            FBInstant.setLoadingProgress(clamped);
        }

        async startGame(): Promise<void> {
            try {
                await FBInstant.startGameAsync();
            } catch (err) {
                console.error('[FBInstantAdapter] startGameAsync failed:', err);
                throw err;
            }
        }

        getPlayerID(): string | null {
            try {
                return FBInstant.player.getID();
            } catch {
                return null;
            }
        }
    }
    ```

    **Step 5 — Write assets/core/scripts/adapters/PlatformDetector.ts.**

    ```typescript
    // PlatformDetector.ts — Runtime adapter factory
    // Called ONCE from Boot.onLoad() before any platform operation.
    // The returned adapter is stored on Boot or a persistent manager and shared globally.
    //
    // Detection logic:
    //   FBInstant global present at runtime → FBInstantAdapter (FB build)
    //   Otherwise                           → NullPlatformAdapter (native / dev preview)

    import { sys } from 'cc';
    import { IPlatformAdapter } from './IPlatformAdapter';
    import { FBInstantAdapter } from './FBInstantAdapter';
    import { NullPlatformAdapter } from './NullPlatformAdapter';

    export function createPlatformAdapter(): IPlatformAdapter {
        // FBInstant is injected as a global by Creator's fb-instant-games build template.
        // sys.isBrowser guards against native builds where globalThis.FBInstant is never defined.
        const isFBInstant = sys.isBrowser && typeof (globalThis as any).FBInstant !== 'undefined';

        if (isFBInstant) {
            console.log('[PlatformDetector] Platform: FB Instant Games');
            return new FBInstantAdapter();
        }

        console.log('[PlatformDetector] Platform: Native / Dev (NullAdapter)');
        return new NullPlatformAdapter();
    }
    ```

    Update Boot.ts to use the platform adapter. The existing stub from Plan 01-01 calls loadBundle directly. Replace the _bootSequence implementation with the correct platform-adapter-aware sequence.

    The sequence order is non-negotiable (see RESEARCH.md Pitfall 4):
    1. createPlatformAdapter() — select adapter based on runtime
    2. adapter.initialize() — FB: initializeAsync(); Native: immediate resolve
    3. adapter.setLoadingProgress(10) — early loading started
    4. Load game-assets bundle (assetManager.loadBundle)
    5. adapter.setLoadingProgress(100) — all assets ready
    6. adapter.startGame() — FB: startGameAsync(); Native: immediate resolve
    7. director.loadScene('Gameplay')

    Read the current Boot.ts content first, then write the updated version:

    ```typescript
    // Boot.ts — Orchestrates platform init and manager persistence
    // Attached to: root node of Boot.scene (first scene loaded)
    //
    // SEQUENCE CONTRACT (must not be reordered):
    //   1. addPersistRootNode — before ANY scene transition
    //   2. adapter.initialize() — before ANY other adapter call
    //   3. loadBundle('game-assets') — between initialize and startGame
    //   4. adapter.setLoadingProgress(100) — after bundle loaded
    //   5. adapter.startGame() — after setLoadingProgress(100), before loadScene
    //   6. director.loadScene('Gameplay')
    //
    // See RESEARCH.md Pitfall 4 for consequences of wrong order.

    import { _decorator, Component, director, game, assetManager } from 'cc';
    import { IPlatformAdapter } from '../../../core/scripts/adapters/IPlatformAdapter';
    import { createPlatformAdapter } from '../../../core/scripts/adapters/PlatformDetector';
    const { ccclass } = _decorator;

    @ccclass('Boot')
    export class Boot extends Component {
        private _platformAdapter!: IPlatformAdapter;

        onLoad(): void {
            // Step 1: Persist this node (and all sibling manager components)
            // across all scene transitions. Must run before any loadScene call.
            game.addPersistRootNode(this.node);

            // Step 2: Select platform adapter (FB Instant or Null)
            this._platformAdapter = createPlatformAdapter();

            // Step 3: Begin async boot sequence
            this._bootSequence().catch(err => {
                console.error('[Boot] Boot sequence failed:', err);
            });
        }

        private async _bootSequence(): Promise<void> {
            // Platform init (FB: initializeAsync; Native: no-op)
            await this._platformAdapter.initialize();
            this._platformAdapter.setLoadingProgress(10);

            // Load game-assets bundle. All art, audio, and gameplay scenes live here.
            // This call returns AFTER the bundle manifest is fetched — actual scene
            // assets are loaded on demand by director.loadScene.
            await new Promise<void>((resolve, reject) => {
                assetManager.loadBundle('game-assets', (err, _bundle) => {
                    if (err) {
                        console.error('[Boot] Failed to load game-assets bundle:', err);
                        reject(err);
                        return;
                    }
                    resolve();
                });
            });

            this._platformAdapter.setLoadingProgress(100);

            // Signal FB Instant that the game is ready (FB: startGameAsync; Native: no-op)
            await this._platformAdapter.startGame();

            // Transition to the Gameplay scene (loaded from game-assets bundle)
            director.loadScene('Gameplay');
        }
    }
    ```
