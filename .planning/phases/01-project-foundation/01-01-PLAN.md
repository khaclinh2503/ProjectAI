**index.html** — replace the entire file content (the template has a different structure):

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />
  <title>Bloom Tap</title>
  <style>
    * {
      margin: 0;
      padding: 0;
      box-sizing: border-box;
    }
    body {
      background: #000;
      overflow: hidden;
    }
    #game-container {
      width: 100%;
      height: 100dvh;
      padding-top: env(safe-area-inset-top);
      padding-bottom: env(safe-area-inset-bottom);
      touch-action: none;
    }
    canvas {
      touch-action: none;
      display: block;
    }
  </style>
</head>
<body>
  <div id="game-container"></div>
  <script type="module" src="/src/main.ts"></script>
</body>
</html>
```

Key requirements enforced here:
- `viewport-fit=cover` — required for env(safe-area-inset-*) to return non-zero values on iPhones
- `touch-action: none` on BOTH #game-container AND canvas — prevents page scroll on tap (pitfall from STATE.md)
- `100dvh` — tracks visible viewport as mobile browser chrome shows/hides
- Safe area padding via CSS env() — avoids notch and home indicator overlap

**src/game/config.ts** — create this file (delete or replace any template equivalent):

```typescript
import Phaser from 'phaser';
import { BootScene } from './scenes/BootScene';
import { GameScene } from './scenes/GameScene';

export const config: Phaser.Types.Core.GameConfig = {
  type: Phaser.AUTO,
  width: 390,
  // height is intentionally omitted — Scale.FIT reads it from #game-container CSS
  parent: 'game-container',
  backgroundColor: '#1a1a2e',
  resolution: window.devicePixelRatio,
  scale: {
    mode: Phaser.Scale.FIT,
    autoCenter: Phaser.Scale.CENTER_BOTH,
  },
  scene: [BootScene, GameScene],
};
```

Critical: `resolution: window.devicePixelRatio` MUST be set here. Retrofitting it later requires full coordinate system recalculation across all scenes. Do NOT omit it.

Both BootScene and GameScene must be in the `scene` array. TypeScript will error on the imports until Plan 02 creates those files — that is expected. If the build fails ONLY because BootScene.ts / GameScene.ts do not exist yet, create empty placeholder exports immediately:

```typescript

  constructor() { super({ key: 'BootScene' }); }
}

// src/game/scenes/GameScene.ts (TEMPORARY — replaced in Plan 02)

export class GameScene extends Phaser.Scene {
  constructor() { super({ key: 'GameScene' }); }
}
```
