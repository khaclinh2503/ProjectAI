  <action>
    There is no CLI for Cocos Creator project creation. The human must use the Cocos Creator 3.8.8 editor to complete these steps.

    Step 1 — Create the project:
    1. Open Cocos Creator 3.8.8 (Dashboard)
    2. Click "New Project"
    3. Select template: "2D" → "Hello World" (TypeScript)
    4. Set project location: E:/workspace/ProjectAI/
    5. Set project name: bloom-tap (or leave as-is if the folder already exists)
    6. Click "Create and Open"

    Step 2 — Configure design resolution:
    1. In the editor, go to Menu → Project → Project Settings
    2. Navigate to "General" tab
    3. Under "Default canvas setting":
       - Design Width: 720
       - Design Height: 1280
       - Fit Width: CHECKED (enabled)
       - Fit Height: UNCHECKED (disabled)
       - Orientation: Portrait
    4. Click "Save"

    Step 3 — Create BootScene:
    1. In the Assets panel, right-click on "assets" folder → "New Folder" → name it "scenes"
    2. Right-click on "scenes" folder → "Create" → "Scene"
    3. Name it "BootScene"
    4. Double-click BootScene to open it
    5. Confirm the scene has a "Canvas" node in Hierarchy. If not: right-click Hierarchy → "Create" → "UI Component" → "Canvas"

    Step 4 — Create GameScene:
    1. Right-click on the "scenes" folder → "Create" → "Scene"
    2. Name it "GameScene"
    3. Double-click GameScene to open it
    4. Ensure it has a Canvas node
    5. Add a Label node: Right-click Canvas → "Create" → "UI Component" → "Label"
    6. With the Label node selected, in Properties panel set String: "Bloom Tap", Color: white (#FFFFFF)
    7. Save the scene (Ctrl+S)

    Step 5 — Create "scripts" folder:
    1. Right-click on "assets" folder → "New Folder" → name it "scripts"

    Step 6 — Clean up template:
    - The Hello World template creates a default scene (named "NewScene" or "HelloWorld") in assets/ root. Delete it — only keep the two scenes in assets/scenes/.

<task type="auto">
  <name>Task 2: Configure tsconfig.json with TypeScript strict mode</name>
  <files>tsconfig.json</files>
  <action>
    Read the existing tsconfig.json that Cocos Creator generated (it extends ./tmp/tsconfig.cocos.json). Replace its contents with the correct strict-mode configuration.

    The file MUST extend "./tmp/tsconfig.cocos.json" — Cocos sets target, module, isolatedModules, and experimentalDecorators there. Do NOT add these fields manually (they conflict with Cocos' internal compiler).

    Only add: strict, skipLibCheck. Do NOT set: target, module, isolatedModules, experimentalDecorators (Cocos controls these — overriding causes IDE/runtime mismatch per RESEARCH.md Pitfall 4).

    Write this exact content:
    ```json
    {
        "extends": "./tmp/tsconfig.cocos.json",
        "compilerOptions": {
            "strict": true,
            "skipLibCheck": true
        }
    }
    ```

    After writing, verify the Cocos Creator editor does not show TypeScript errors in the bottom panel (a few seconds after saving). Strict-mode warnings on the generated Hello World template files are acceptable — they will be replaced in Plan 02.
  </action>
