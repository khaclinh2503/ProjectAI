## Design System

| Property | Value |
|----------|-------|
| Tool | none — Cocos Creator native rendering |
| Preset | not applicable |
| Component library | Cocos Creator Graphics + Label + UIOpacity + tween |
| Icon library | none — programmatic Graphics drawing |
| Font | Cocos Creator default system font (existing project convention) |

**Note:** This is a Cocos Creator mobile game, not a web/React project. shadcn gate does not apply. All UI is rendered via Cocos Creator's Graphics API, Label components, and tween animations. No CSS, no HTML.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/label gaps inside HUD slots |
| sm | 8px | Padding inside PowerUpHUD slots |
| md | 16px | Gap between HUD elements |
| lg | 24px | Section separation (HUD from grid) |
| xl | 32px | Layout margins |
| 2xl | 48px | Major vertical separation |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Screen shake displacement: 8px on X axis (fits xs–sm range; interaction-specific, not layout spacing)
- Touch targets: 44px minimum (Cocos Creator node UITransform content size — existing game convention)

## Typography

All text is Cocos Creator Label components. Font size values are integers in Cocos px units.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Score float (small) | 32px | regular (400) | 1.2 |
| Score float (large) | 56px | regular (400) | 1.2 |
| Combo label | 28px | semibold (600) | 1.2 |
| HUD timer / score | 20px | regular (400) | 1.5 |

**Score float range:** 32px–56px, driven by `getFloatFontSize(rawScore)` — existing formula, no change. Source: JuiceHelpers.ts.

**Combo label (FIX-01):**
- Format: `x{streak}` — e.g., "x5", "x12" (no "Combo" prefix per D-06)
- Hidden when streak < 2 (D-02): `node.active = false`
- Normal pulse: scale 1.0 → 1.25 → 1.0 over 180ms (existing `_pulseComboLabel()` — no change)
- Milestone pulse (D-05, at multiples of 10: x10, x20, x30...): scale 1.0 → 1.55 → 1.0 over 220ms, brighter flash color (see Color section)

**Score float with power-up multiplier (D-18):**
- Format: `+{score} ×{powerUpMultiplier}` when powerUpMultiplier > 1 — e.g., "+240 ×3"
- Same font size formula applies to the base score value
- The `×{n}` suffix appended in the same label string, no separate node needed

## Interaction Contracts

### FIX-01: Combo Label Display

**Trigger:** Every correct tap that increments streak.

**Normal tap (streak 2–9, 11–19...):**
- Update label string to `x{streak}`
- Run `_pulseComboLabel()`: scale 1.0 → 1.25 → 1.0 in 180ms, easing cubicOut/cubicIn (existing)
- Label color: white Color(255, 255, 255, 255)

**Milestone tap (streak is multiple of 10: 10, 20, 30...):**
- Update label string to `x{streak}`
- Run enhanced pulse: scale 1.0 → 1.55 → 1.0 in 220ms, easing cubicOut/cubicIn
- Flash label color to Color(255, 200, 0, 255) bright gold, then return to white
- Use `Tween.stopAllByTarget(labelNode)` before starting tween (established pattern)

**Streak reset (wrong tap or combo break):**
- Set `node.active = false` — label disappears immediately
- Do NOT show "x1"

**Session start:**
- `node.active = false` — label is hidden from frame 0

### FIX-02: Screen Shake on Wrong Tap

**Trigger:** `handleWrongTap()` in GameController — covers empty cell tap AND BUD/WILTING/DEAD flower tap (D-09).

**Target node:** `gridRenderer.node` (D-07)

**Shake parameters (D-08, Claude's discretion for easing):**
- Displacement: ±8px on X axis only
- Duration: 200ms total
- Oscillations: 3 steps (right → left → right → origin)
- Easing: `sineOut` per step for natural deceleration
- Step timing: 50ms right (+8px), 50ms left (-8px), 50ms right (+8px), 50ms return to Vec3(0,0,0)

**Guard:** Call `Tween.stopAllByTarget(this.gridRenderer.node)` before starting shake tween (established pattern D-10).

**Return to origin:** Final tween step must explicitly return node to Vec3(0, 0, 0) — no residual offset.

### D-21–D-24: Grid Border Glow

**Trigger:** Power-up state change — draw/redraw each frame in `GridRenderer.update()` after `_paintState()` call.

**Implementation:** Graphics overlay drawn on the GridRenderer node (child or reuse existing flashGraphics container — Claude's discretion per CONTEXT.md).

**Border specification (Claude's discretion for width and alpha per CONTEXT.md):**
- Border width: 4px stroke
- Border alpha: 200/255 (78%) — "subtle but visible" per specifics note
- Draw rect matching grid UITransform content size, inset by 2px on each side
- No rounded corners (matches existing cell drawing style)

**Color lookup:** Use table in Color section above (D-22).

**Expiry behavior (D-24):** Border disappears immediately when `powerUpState.isAnyActive(nowMs)` returns false — no fade, alpha goes to 0 in the same frame.

### D-18–D-20: Score Float with Power-Up Multiplier

**Trigger:** Correct tap while powerUpMultiplier > 1.

**Label string format:**
- `+{score} ×{powerUpMultiplier}` — e.g., "+240 ×3"
- If powerUpMultiplier === 1: show `+{score}` only (existing behavior, no change)
- The `×` character is the multiplication sign (U+00D7), not the letter "x"

**Font size:** Unchanged — `getFloatFontSize(rawScore)` applies to the whole label (D-20).

**Color:** White Color(255, 255, 255, 255) — same as positive score float.

**Data flow (D-19):** `handleCorrectTap()` return object must include `powerUpMultiplier` field so GridRenderer can render it. Add to existing `{ flashColor, rawScore, multiplier, isFullBloom }` object.
